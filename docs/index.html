What it is무엇인가

Your localhost design, on the small screen.로컬호스트의 디자인을, 작은 화면으로.

The Open Design (OD) desktop app serves its web UI on 127.0.0.1 only, and the port changes on every restart — so a phone can't reach it. odpeek auto-detects the live OD port and exposes it through one of two paths. It only ever exposes the OD running on your own machine. Open Design(OD) 데스크톱 앱은 웹 UI를 127.0.0.1에만 띄우고 재시작마다 포트가 바뀌어 폰에서 닿을 수 없다. odpeek은 현재 OD 포트를 자동 감지해 두 방식 중 하나로 노출한다. 노출되는 건 오직 본인 컴퓨터의 OD뿐이다.

private · tailnet

up — Tailscale

Visible to your own devices only (your phone on your tailnet). Never the public internet.내 기기에만 보임(내 tailnet의 폰). 공개 인터넷에 안 열림.

public · authenticated

tunnel — Cloudflare

A public HTTPS URL guarded by Basic auth. Reachable by anyone you hand the URL + password to.Basic 인증으로 보호된 공개 HTTPS URL. URL+비번을 받은 사람만 접속.

  • Works on cellular / any network셀룰러·어떤 망에서든 동작
  • Dials out to localhost → bypasses firewall / CGNAT / DNS로컬호스트로 아웃바운드 → 방화벽·CGNAT·DNS 우회
  • No Cloudflare account needed for the quick tunnel빠른 터널은 Cloudflare 계정 불필요

Security보안

A public URL, hardened.공개 URL, 단단하게.

The tunnel URL is public, so the auth proxy ships hardened by default — but it's still a public exposure you should manage.tunnel URL은 공개이므로 인증 프록시가 기본 하드닝돼 있다 — 그래도 공개 노출이니 사용자가 관리해야 한다.

brute-force lockout
8 fails → 15-min block + global throttle8회 실패 → 15분 차단 + 전역 스로틀
auth bypass
WebSocket handshake verified before piping파이프 전 WebSocket 핸드셰이크 검증
headers
HSTS, nosniff, frame & referrer policy; auth header stripped upstreamHSTS·nosniff·프레임/리퍼러 정책, 인증 헤더 미전달
credentials
timing-safe compare; ~71-bit random default; passed via env not argv타이밍 안전 비교, ~71비트 난수 기본, argv 대신 env 전달
idle shutdown
tunnel auto-closes after 30 min idle30분 유휴 시 자동 종료
logs
attempts logged to ~/.odpeek/auth.log (sanitized, never the password)~/.odpeek/auth.log에 기록(살균, 비번 제외)

⚠ The tunnel URL is public터널 URL은 공개

Run odpeek off when you're done, and be mindful of exposing sensitive designs. Don't share the URL + password loosely.다 봤으면 odpeek off, 민감한 디자인 노출에 주의. URL+비번을 함부로 공유하지 말 것.

✓ No inbound needed인바운드 불필요

If macOS asks "Allow incoming connections for node?", you can Deny — the design relies on loopback + outbound.macOS가 "node 인바운드 허용?"을 물으면 거부해도 된다 — 설계가 루프백+아웃바운드 기반이다.
